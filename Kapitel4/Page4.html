DBSCAN Clustering

DBSCAN – dichtebasiertes Clustering

  • DBSCAN = Density-based Spatial Clustering of Applications with Noise
  • Grenzt dichte von weniger dichten Regionen ab → beliebige Clusterformen möglich
  • Noise-Punkte erhalten das Label -1 und werden als eigene „Klasse" behandelt
  • Die DBSCAN-Klasse aus sklearn hat keine .predict()-Methode – dies ist bewusst so gestaltet
  • Für die Zuordnung neuer Datenpunkte kann z.B. ein k-Nearest-Neighbors Klassifikator genutzt werden
  • Wichtig: Hyperparameter-Suche im 2D-Raum (eps × min_samples) ist nötig
BegriffBedeutung
epsRadius der Epsilon-Nachbarschaft
min_samplesMindestanzahl Punkte für Core-Region
Core RegionAusreichend dichte Region
Density-reachableÜber Core-Punkte erreichbar
NoisePunkte außerhalb aller Cluster; Label -1
 
from sklearn.datasets import make_moons
from sklearn.cluster import DBSCAN
import numpy as np, matplotlib.pyplot as plt

X, y = make_moons(n_samples=1000, noise=0.05, random_state=42)

# Zu kleines eps → zu viele Fragmente
dbscan = DBSCAN(eps=0.05, min_samples=5); dbscan.fit(X)
print(dbscan.labels_[:10])          # enthält -1 für Noise
print(np.unique(dbscan.labels_))    # z.B. [-1  0  1  2  3  4  5  6]

# Korrektes eps
dbscan = DBSCAN(eps=0.20, min_samples=5); dbscan.fit(X)
plt.scatter(X[:, 0], X[:, 1], c=dbscan.labels_, s=50, cmap='viridis');
DBSCAN hat keine .predict()-Methode. Für neue Punkte → kNN-Klassifikator nutzen.
VorteileNachteile
Clusteranzahl muss nicht vorab festgelegt werdenNicht vollständig deterministisch (Border-Points)

Nur 2 Hyperparametereps und min_samples nicht universell

Neue Punkte zu DBSCAN-Clustern zuordnen

  • Lösung: k-Nearest-Neighbors (kNN) Klassifikator wird auf den Core-Punkten des DBSCAN trainiert
  • Die Klassifizierung eines neuen Datenpunktes geschieht über ein Majoritätsvotum der k nächsten Nachbarn
  • Das „Lernen" des kNN besteht nur aus dem Abspeichern des Datensatzes und dessen Labels
  • Labels kommen aus dem DBSCAN-Training: dbscan.labels_[dbscan.core_sample_indices_]
from sklearn.datasets import make_moons
from sklearn.cluster import DBSCAN
from sklearn.neighbors import KNeighborsClassifier
import matplotlib.pyplot as plt

X, y = make_moons(n_samples=1000, noise=0.05, random_state=42)
dbscan = DBSCAN(eps=0.20, min_samples=5); dbscan.fit(X)

knn = KNeighborsClassifier(n_neighbors=50)
knn.fit(dbscan.components_, dbscan.labels_[dbscan.core_sample_indices_])

new_point_1 = [[-0.5, 0.5]]
new_point_2 = [[2, -0.5]]
print(knn.predict(new_point_1))   # [1]
print(knn.predict(new_point_2))   # [0]

Anwendungsbeispiel: Brownfield-Maschinen

Ältere Maschinen ohne Zustandsausgabe → Betriebszustände datengetrieben bestimmen (Weihenstephaner Standard).

  • Im Maschinenbau gibt es den sog. Weihenstephaner Standard, in dem verschiedene Maschinenzustände kodiert sind
  • Bei neueren Maschinen gibt es Schnittstellen, die diese Zustände ausgeben können
  • Bei älteren Maschinen (Brownfield / Retrofits) fehlen diese Schnittstellen oft
  • Ziel: datengetrieben Betriebszustände aus Vibrations- und Temperatursensordaten (1-minütige Snapshots) bestimmen
  • Vorgehen: k-means Clustering auf Sensordaten anwenden und Betriebszustände den Clustern zuordnen
Erkenntnis: Scaling kann helfen – aber es kann nicht pauschal behauptet werden, dass es immer angewendet werden muss!

import numpy as np
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
from yellowbrick.cluster import SilhouetteVisualizer
import matplotlib.pyplot as plt

X, y = make_blobs(n_samples=3000, n_features=2, centers=6, cluster_std=0.55, random_state=2)
X[:, 0] = np.abs(X[:, 0])      # Vibration
X[:, 1] = np.abs(X[:, 1]) * 7  # Temperatur
np.save('machine_state.npy', X)

X = np.load('machine_state.npy')
scaler = StandardScaler()
X = scaler.fit_transform(X)

list_of_k = range(2, 14)
list_of_inertias, list_of_scores = [], []
fig, ax = plt.subplots(6, 2, figsize=(15, 30))
for n_clusters in list_of_k:
    kmeans = KMeans(n_clusters)
    cluster_labels = kmeans.fit_predict(X)
    list_of_inertias.append(kmeans.inertia_)
    list_of_scores.append(silhouette_score(X, cluster_labels))
    visualizer = SilhouetteVisualizer(kmeans, colors='yellowbrick', ax=ax.flat[n_clusters - 2])
    visualizer.fit(X)
    ax.flat[n_clusters - 2].set_title(f'k: {n_clusters}')

plt.figure(); plt.plot(list_of_k, list_of_inertias, 'go--'); plt.ylabel('Inertia'); plt.xlabel('k')
plt.figure(); plt.plot(list_of_k, list_of_scores, 'go--'); plt.ylabel('Silhouette'); plt.xlabel('k')

Vergleich aller Algorithmen

AlgorithmusVorteileNachteile
k-means Schnell, einfach, breit anwendbar Initialisierungsabhängig, nur lineare Trennung, Ausreißer problematisch
k-means++ Verbesserte Initialisierung, konvergiert schneller Wie k-means, aber stabiler
k-Median Robuster gegenüber Ausreißern Langsamer als k-means
Spectral Clustering Nicht-lineare Strukturen möglich Rechenintensiv, Hyperparameter nötig
DBSCAN Beliebige Formen, Noise-Behandlung, k nicht nötig eps/min_samples schwer zu wählen, unterschiedliche Dichten problematisch
