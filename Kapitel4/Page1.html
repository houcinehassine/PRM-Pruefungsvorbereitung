k-means – Grundidee

  • Wir beginnen mit dem k-means Algorithmus
  • Der k-means Algorithmus sucht nach einer vorabbestimmten Anzahl an Clustern in einem nicht gelabelten Datensatz
  • Heuristik des k-means Algorithmus:
    • Der Cluster-Mittelpunkt (== Centroid) ist das arithmetische Mittel aller Punkte des jeweiligen Clusters
    • Jeder Datenpunkt ist näher an seinem eigenen Centroid als an anderen
import numpy as np, matplotlib.pyplot as plt
from sklearn.datasets import make_blobs

X, y_true = make_blobs(n_samples=300, centers=4, cluster_std=0.60, random_state=0)
plt.scatter(X[:, 0], X[:, 1], s=50);

k-means in sklearn

  • In sklearn weist der k-means Algorithmus die gleiche Struktur wie andere Modelle auf
  • Als wichtiger Zusatz bei der Instanziierung muss die Anzahl an Clustern angegeben werden (n_clusters)
  • Die Zugehörigkeit der Datenpunkte erhalten wir über die .predict()-Methode
  • Die erlernten Centroids stecken im .cluster_centers_-Attribut des Modells
from sklearn.cluster import KMeans

kmeans = KMeans(n_clusters=4)
kmeans.fit(X)
y_kmeans = kmeans.predict(X)        # Labels
centers  = kmeans.cluster_centers_  # Centroids

Expectation-Maximization (EM)

  • Die Heuristik des k-means wird als sog. Expectation-Maximization-Algorithmus umgesetzt
  • Der EM-Algorithmus findet in vielen Machine Learning Modellen Anwendung
  • Es handelt sich um einen iterativen Algorithmus
  • Generelle Idee:
    • Initialisiere eine zufällige Konfiguration des Modells
    • E(xpectation)-Schritt: Zuordnung der Daten zu den einzelnen Teilen des Modells
    • M(aximization)-Schritt: Parameter an die neueste Zuordnung anpassen
    • Konvergenz: findet keine wesentliche Verbesserung mehr statt → Abbruch
  • Expectation beim k-means: unsere „Erwartung" updaten, zu welchem Cluster die Datenpunkte gehören
  • Maximization beim k-means: ein Gütemaß wird maximiert (durch Berechnung des Mittelwerts)
EM-Schritt Bedeutung bei k-means
Initialisierung Centroids zufällig wählen
E-Schritt Datenpunkte dem nächsten Centroid zuweisen
M-Schritt Centroids als Mittelwert neu berechnen
Konvergenz Abbruch ohne wesentliche Verbesserung

Formaler Ablauf von k-means

Input / Output

  • Input:
    • \(\mathbf{X}\): Datenmatrix mit Dimensionen \(N \times T\)
    • \(N\): Anzahl der Beobachtungen
    • \(T\): Anzahl der Features (z.B. \(\bar{x}, \sigma^2, \ldots\))
    • \(k\): Anzahl zu extrahierender Cluster
  • Output:
    • \(\mathbf{c}_z\): Cluster-Centroids mit \(z = 1, \ldots, k\)
    • \(\hat{z}_t\): zugewiesener Cluster-Index des Datenpunkts \(\mathbf{x}_t\)

Schritt 1 – Initialisierung

  • Centroids \(\mathbf{c}_z\) werden zufällig gewählt

Schritt 2 – E-Schritt: Zuordnung

\[ \hat{z}_t = \arg\min_z \bigl( d(\mathbf{c}_z,\, \mathbf{x}_t) \bigr) \]

Schritt 3 – M-Schritt: Centroid-Update

\[ \mathbf{c}_z \leftarrow \frac{\sum_{t=1}^{T} u(t,z,\hat{z}_t)\,\mathbf{x}_t}{\sum_{t=1}^{T} u(t,z,\hat{z}_t)} \]

mit der Indikatorfunktion \(u(t, z, \hat{z}_t) = 1\) falls \(\hat{z}_t = z\), sonst \(0\).

Schritt 4 – Wiederholen bis Konvergenz

  1. Zufällige Initialisierung der Centroids.
  2. Zuordnung jedes Punkts zum nächsten Centroid.
  3. Berechnung neuer Centroids.
  4. Wiederholung bis zur Konvergenz.
