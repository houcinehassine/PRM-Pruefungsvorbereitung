k-means from Scratch

import numpy as np
from scipy.spatial.distance import cdist
from sklearn.datasets import make_blobs

X, _ = make_blobs(n_samples=300, centers=4, cluster_std=0.60, random_state=0)

def find_clusters(X, num_clusters, threshold=0.0001):
    centroids = X[np.random.permutation(X.shape[0])[:num_clusters]]
    deviation = threshold + 1

    while deviation > threshold:
        distances = cdist(X, centroids)
        labels = np.argmin(distances, axis=1)
        new_centers = np.array([X[labels == n].mean(0) for n in range(num_clusters)])
        deviation = np.sum((new_centers - centroids) ** 2)
        centroids = new_centers

    return centroids, labels

centroids, labels = find_clusters(X, 4)

Schwächen von k-means / EM

  • Kein globales Optimum garantiert – abhängig von Initialbedingungen.
  • Mehrfache Initialisierungen nötig → n_init in sklearn (standardmäßig aktiv).
  • Anzahl Cluster muss vorab festgelegt werden.
  • Nur für linear separierbare Strukturen geeignet.
  • Ausreißer beeinflussen das Ergebnis stark.

Varianten von k-means

Spectral Clustering

  • Mittels graphentheoretischer Maße werden die Daten in einen höherdimensionalen Raum projiziert
  • In diesem höherdimensionalen Raum kann k-means auch mit linearen Grenzen Cluster separieren, die im niedrigdimensionalen Raum nur nicht-linear separierbar wären
from sklearn.cluster import SpectralClustering
model = SpectralClustering(n_clusters=2, affinity='nearest_neighbors', assign_labels='kmeans')
labels = model.fit_predict(X)

Vorteile und Nachteile von k-means

Vorteile Nachteile

Leicht verständlich Kein globales Optimum garantiert
