4.1 Loss Function

Durch die mathematische Herleitung erhält man für die Soft-Margin-SVM eine gut nutzbare Kostenfunktion, die sich direkt in Python implementieren lässt [web:41][web:35].

\[ \min_{\mathbf{w}, b} \frac{1}{2}\|\mathbf{w}\|^2 + C \sum_i \max \{0, 1 - y_i(\langle \mathbf{w}, \mathbf{x}_i \rangle + b)\} \]

Der erste Term ist die Regularisierung, der zweite Term ist die Hinge Loss [web:41][web:35].

Import & Daten

Für ein einfaches Beispiel werden künstliche Daten mit make_blobs erzeugt, anschließend mit StandardScaler skaliert und die Labels von \(\{0,1\}\) nach \(\{-1,1\}\) umkodiert [web:50][web:51]. 

import matplotlib.pyplot as plt
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler


# Data and parameters
X, y = make_blobs(n_samples=1000, centers=2, cluster_std=0.1)
X = StandardScaler().fit_transform(X)
y = 2*y - 1  # convert 0,1 to -1,1


w = np.random.rand(X.shape[1])
b = np.random.rand()

Loss Function in Python

Die Hinge Loss lautet \( \max(0, 1 - y \cdot f(\mathbf{x})) \), wobei \( f(\mathbf{x}) = \langle \mathbf{w}, \mathbf{x} \rangle + b \) die lineare Entscheidungsfunktion ist [web:35][web:33]. 

def svm_loss_function(X, y, w, b, C):
    penalties = 1 - y * np.dot(X, w) - b
    penalties[penalties < 0] = 0  # equivalent to max{0, distance} = hinge_loss
    hinge_loss = C * np.mean(penalties)  # scaling with 1/m for independence of data size
    loss = 1/2 * np.dot(w, w) + hinge_loss
    return loss


svm_loss_function(X, y, w, b, 1)

Punkte mit negativem Penalty tragen nichts zur Hinge Loss bei, während Punkte innerhalb des Margins oder auf der falschen Seite bestraft werden [web:35][web:33].

4.2 Gradienten

Um die Kostenfunktion zu minimieren, benötigt man die Gradienten nach \(\mathbf{w}\) und \(b\). Aktiv sind dabei nur Datenpunkte, für die der Margin-Ausdruck größer als 0 ist [web:33][web:41]. 

def svm_gradients(X, y, w, b, C):
    m = X.shape[0]
    margins = 1 - y * (X.dot(w) + b)
    active = margins > 0  # indices where hinge loss is active
    dw = w - C/m * X[active].T.dot(y[active])  # scaling with 1/m
    db = -C/m * np.sum(y[active])
    return dw, db


svm_gradients(X, y, w, b, 1)

Der Term \(dw\) enthält sowohl den Regularisierungsteil \(\mathbf{w}\) als auch den Beitrag der aktiven Hinge-Loss-Punkte [web:41][web:33].

4.3 Gradientenabstieg – Algorithmus

Beim Gradientenabstieg werden die Parameter iterativ in Richtung des negativen Gradienten aktualisiert.

\[ \Delta \mathbf{w}^{(i)} = - \eta \nabla \Lambda \mid_{\mathbf{w}^{(i)}} \]
\[ \mathbf{w}^{(i+1)} = \mathbf{w}^{(i)} + \Delta \mathbf{w}^{(i)} \]

Dabei ist \(\eta\) die Lernrate. 

def svm_train(X, y, w, b, C, lr, epochs):
    list_of_costs = []
    for epoch in range(epochs):
        dw, db = svm_gradients(X, y, w, b, C)
        w -= lr * dw
        b -= lr * db
        list_of_costs.append(svm_loss_function(X, y, w, b, C))
    return w, b, list_of_costs


# Anwenden
w, b, list_of_costs = svm_train(X, y, w, b, 1, 0.01, 1000)


plt.plot(list_of_costs)
plt.xlabel('Epochs')
plt.ylabel('Loss')

Wenn das Training funktioniert, sollte der Loss über die Epochen typischerweise sinken, weil die Parameter schrittweise verbessert werden [web:41].

4.4 Visualisierung

Nach dem Training kann man die gelernte Hyperebene in den Feature-Raum einzeichnen. Für eine stabile geometrische Interpretation wird \(\mathbf{w}\) dabei normalisiert [web:41]. 

def plot_decision_line(w, b, x_range):
    fig, ax = plt.subplots(figsize=(6,5))
    xmin, xmax = x_range
    xs = np.linspace(xmin, xmax, 400)
    norm = np.linalg.norm(w)
    w = w / norm
    b = b / norm
    ys = -w[0] * xs - b / w[1]
    ax.plot(xs, ys, linewidth=2, label='decision line')
    ax.set_xlabel('x1')
    ax.set_ylabel('x2')
    ax.legend()
    ax.grid(True)
    return ys


ys = plot_decision_line(w, b, x_range=(X[:, 0].min(), X[:, 0].max()))
plt.scatter(X[:, 0], X[:, 1], c=y)

Die Gerade trennt die beiden Klassen entsprechend der gelernten Parameter \(w\) und \(b\). Ein möglicher Bonus ist eine Funktion animate_svm_training, die Zwischenstände speichert und den Lernprozess dynamisch zeigt.
