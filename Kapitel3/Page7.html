Lineare SVR

Die Support Vector Regression arbeitet mit einer \(\varepsilon\)-Tube: Vorhersagen innerhalb dieses Toleranzbandes werden nicht bestraft.

In sklearn kann man dafür LinearSVR verwenden. Die Parameter \(\varepsilon\) und \(C\) steuern die Breite der Tube und die Stärke der Regularisierung. 

import numpy as np
import matplotlib.pyplot as plt


np.random.seed(42)
m = 50
X = 2 * np.random.rand(m, 1)
y = 4 + 3 * X + np.random.randn(m, 1)


from sklearn.svm import LinearSVR


svmreg1 = LinearSVR(epsilon=1.5, random_state=42)
svmreg2 = LinearSVR(epsilon=0.5, random_state=42)


svmreg1.fit(X, y.ravel())
svmreg2.fit(X, y.ravel())

Support Vectors finden

Bei der Regression sind die Support Vectors die Punkte außerhalb der \(\varepsilon\)-Tube. Sie tragen aktiv zum Modell bei. 

def find_support_vectors(svmreg, X, y):
    ypred = svmreg.predict(X)
    off_margin = np.abs(y - ypred) > svmreg.epsilon
    return np.argwhere(off_margin)


svmreg1.support_ = find_support_vectors(svmreg1, X, y)
svmreg2.support_ = find_support_vectors(svmreg2, X, y)

SVR plotten mit Epsilon

Die Vorhersage wird zusammen mit den \(\varepsilon\)-Grenzen dargestellt. Ein Pfeil im Graphen markiert die Distanz \(\varepsilon\) explizit [web:85][web:96]. 

def plotsvm_regression(svmreg, X, y, axes=None):
    x1s = np.linspace(axes[0], axes[1], 100).reshape(100, 1)
    ypred = svmreg.predict(x1s)
    plt.plot(x1s, ypred, 'k-', linewidth=2, label=r'$\hat{y}$')
    plt.plot(x1s, ypred + svmreg.epsilon, 'k--')
    plt.plot(x1s, ypred - svmreg.epsilon, 'k--')
    plt.scatter(X[svmreg.support_], y[svmreg.support_], s=180, facecolors='#FFAAAA')
    plt.plot(X, y, 'b.')
    plt.xlabel(r'$x_1$', fontsize=18)
    plt.legend(loc='upper left', fontsize=18)
    plt.axis(axes)

Für kleine \(\varepsilon\) werden mehr Punkte als Support Vectors markiert, für größere \(\varepsilon\) entsprechend weniger. 

fig, axes = plt.subplots(ncols=2, figsize=(9, 4), sharey=True)
plt.sca(axes[0])
plotsvm_regression(svmreg1, X, y, axes=[0, 2, 3, 11])
plt.title(r'$\varepsilon = {:.1f}$'.format(svmreg1.epsilon), fontsize=18)
plt.ylabel(r'$y$', fontsize=18, rotation=0)


# Epsilon-Pfeil einzeichnen
eps_x1 = 1
eps_ypred = svmreg1.predict([[eps_x1]])
plt.plot([eps_x1, eps_x1], [eps_ypred, eps_ypred - svmreg1.epsilon], 'k-', linewidth=2)
plt.annotate('', xy=(eps_x1, eps_ypred), xycoords='data',
             xytext=(eps_x1, eps_ypred - svmreg1.epsilon), textcoords='data',
             arrowprops=dict(arrowstyle='-', linewidth=1.5))
plt.text(0.91, 5.6, r'$\varepsilon$', fontsize=20)


plt.sca(axes[1])
plotsvm_regression(svmreg2, X, y, axes=[0, 2, 3, 11])
plt.title(r'$\varepsilon = {:.1f}$'.format(svmreg2.epsilon), fontsize=18)
plt.show()

Nicht-lineare SVR: Poly-Kernel

Für nicht-lineare Zusammenhänge kann ein Polynomial Kernel verwendet werden. Der Grad und der Regularisierungsparameter \(C\) beeinflussen die Form der Regression [web:89][web:96]. 

from sklearn.svm import SVR


# Daten erzeugen
np.random.seed(42)
m = 100
X = 2 * np.random.rand(m, 1) - 1
y = (0.2 + 0.1 * X + 0.5 * X**2 + np.random.rand(m, 1)).ravel()


plt.scatter(X, y)


# Poly Kernel
svmpolyreg1 = SVR(kernel='poly', degree=2, C=100, epsilon=0.1, gamma='scale')
svmpolyreg2 = SVR(kernel='poly', degree=2, C=0.01, epsilon=0.1, gamma='scale')
svmpolyreg1.fit(X, y)
svmpolyreg2.fit(X, y)


fig, axes = plt.subplots(ncols=2, figsize=(9, 4), sharey=True)
plt.sca(axes[0])
plotsvm_regression(svmpolyreg1, X, y, axes=[-1, 1, 0, 1])
plt.title(r'Poly: degree={}, C={:.2f}, $\varepsilon={:.1f}$'.format(
    svmpolyreg1.degree, svmpolyreg1.C, svmpolyreg1.epsilon), fontsize=18)
plt.sca(axes[1])
plotsvm_regression(svmpolyreg2, X, y, axes=[-1, 1, 0, 1])
plt.title(r'Poly: degree={}, C={:.2f}, $\varepsilon={:.1f}$'.format(
    svmpolyreg2.degree, svmpolyreg2.C, svmpolyreg2.epsilon), fontsize=18)
plt.show()

Nicht-lineare SVR: RBF-Kernel

Der RBF-Kernel ist eine weitere nicht-lineare Variante. Auch hier steuern \(C\) und \(\varepsilon\) die Flexibilität der Regression [web:85][web:96][web:98]. 

# RBF Kernel
svmreg1 = SVR(kernel='rbf', C=100, epsilon=0.1)
svmreg2 = SVR(kernel='rbf', C=0.01, epsilon=0.1)


svmreg1.fit(X, y)
svmreg2.fit(X, y)


fig, axes = plt.subplots(ncols=2, figsize=(9, 4), sharey=True)
plt.sca(axes[0])
plotsvm_regression(svmreg1, X, y, axes=[-1, 1, 0, 1])
plt.title(r'RBF: C={:.2f}, $\varepsilon={:.1f}$'.format(svmreg1.C, svmreg1.epsilon), fontsize=18)
plt.sca(axes[1])
plotsvm_regression(svmreg2, X, y, axes=[-1, 1, 0, 1])
plt.title(r'RBF: C={:.2f}, $\varepsilon={:.1f}$'.format(svmreg2.C, svmreg2.epsilon), fontsize=18)
plt.show()

Vor- und Nachteile von SVMs

Support Vector Machines besitzen einige klare Stärken, aber auch wichtige Grenzen, die man bei der Modellwahl beachten sollte.

Pros

Vorteil Erklärung
Höherdimensionale Datensätze SVMs sind sehr effektiv bei hochdimensionalen Daten [file:23].
Dimensionen > Datenpunkte Sie helfen besonders in Fällen, in denen die Anzahl der Dimensionen größer ist als die Anzahl der Datenpunkte [file:23].
Komputativ effizient Besonders bei kleinen bis mittleren Datensätzen gelten SVMs als effizient [file:23].
Geometrische Interpretation SVMs lassen sich pädagogisch gut über Hyperebene, Margin und Support Vectors erklären [file:23].

Cons

Nachteil Erklärung
Sehr große Datensätze Bei sehr vielen Datenpunkten bekommen SVMs Probleme, insbesondere weil die Kernel-Matrix zu groß werden kann [file:23].
Stark überlappende Klassen Viel Rauschen und starke Überlappung führen zu einer schlechteren Trennung der Klassen [file:23].
Probabilistische Interpretation Eine direkte Wahrscheinlichkeitsausgabe ist bei SVMs nicht vorgesehen [file:23].

Kurze Zusammenfassung

SVMs: gut bei kleiner Datenmenge und hoher Dimension, schwächer bei großen Datensätzen und viel Rauschen. Keine probabilistische Ausgabe, aber gute geometrische Interpretierbarkeit [file:23].
