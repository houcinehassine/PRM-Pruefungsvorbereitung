Nicht-lineare Probleme

Im bisherigen Digits-Beispiel wurde implizit angenommen, dass die Klassen linear trennbar sind.

Falls keine lineare Hyperebene beide Klassen trennen kann, liegt ein nicht-linear separierbares Problem vor.

SVMs können auch solche Probleme lösen, indem die Daten in einen höherdimensionalen Raum projiziert werden [web:59][web:61].

Radial Basis Functions

Eine Möglichkeit zur Projektion ist die Verwendung von Radial Basis Functions (RBFs). Das sind skalare Funktionen \(\phi(r)\), die nur vom Abstand zum Ursprung abhängen [web:61][web:62].

\[ r = \exp\left(-\sum_j x_j^2\right) \]

Dadurch wird aus einem 2D-Datensatz ein Datensatz in einer höheren Dimension. In dieser höheren Dimension kann das Problem wieder linear separierbar werden [web:61][web:62].

Beispiel mit make_circles

Für das Beispiel werden nicht-linear separierbare Daten mit de>make_circles erzeugt und zunächst mit einem linearen SVC modelliert [web:45]. 

de>from sklearn.datasets import make_circles
from sklearn.svm import SVC


# Nicht linear separierbare Daten erzeugen
X, y = make_circles(100, factor=.1, noise=.1)
clf = SVC(kernel='linear').fit(X, y)


plt.scatter(X[:, 0], X[:, 1], c=y, s=50, cmap='autumn')
plot_svc_decision_function(clf, plot_support=False)

Mit einem linearen Kernel bleibt die Trennung im 2D-Raum unzureichend [web:59][web:61].

Projektion in höhere Dimensionen

Der RBF-Term wird als zusätzliche Dimension an die Merkmale angehängt. Damit entsteht ein erweitertes Feature-Set, das im Beispiel als \(X_{5d}\) gespeichert wird. 

de># RBF Projektion zeigen
r = np.exp(-(X**2).sum(axis=1))
X_5d = np.hstack([X, r.reshape(-1, 1)])
print(X_5d[:5])

Die Funktion de>plot_3D zeigt diese Projektion im 3D-Raum: Die Klassen werden dort deutlich besser voneinander getrennt [web:61][web:62]. 

de>def plot_3D(elev=30, azim=30, X=X, y=y):
    ax = plt.subplot(projection='3d')
    r = np.exp(-(X**2).sum(axis=1))
    ax.scatter(X[:, 0], X[:, 1], r, c=y, s=50, cmap='autumn', edgecolors='gray')
    ax.view_init(elev=elev, azim=azim)
    ax.set_xlabel('x')
    ax.set_ylabel('y')
    ax.set_zlabel('r')


plt.figure(figsize=(10, 7))
plot_3D()

Kernel-Trick

Der Kernel-Trick geht noch weiter: Er fügt die zusätzliche Dimension implizit hinzu, ohne die Projektion explizit auszurechnen [web:59][web:62].

In diesem höherdimensionalen Raum wird eine linear separierende Hyperebene gesucht. Im ursprünglichen Raum erscheint diese dann als nicht-lineare Entscheidungsgrenze.

In de>sklearn wird das über de>kernel='rbf' umgesetzt [web:38][web:65]. 

de>clf = SVC(kernel='rbf', C=1E6)
clf.fit(X, y)


plt.scatter(X[:, 0], X[:, 1], c=y, s=50, cmap='autumn', edgecolors='gray')
plot_svc_decision_function(clf)
plt.scatter(clf.support_vectors_[:, 0], clf.support_vectors_[:, 1],
            s=300, lw=1, facecolors='none')

Multi-Class SVMs

Liegen mehr als zwei Klassen vor, kann der binäre SVM-Ansatz nicht direkt eins-zu-eins angewandt werden.

Eine Möglichkeit ist one-vs-one: Für jede Klassenpaarung wird ein eigener Klassifikator trainiert [web:38][web:60][web:63].

Für \(N\) Klassen müssen dann

\[ \frac{N(N-1)}{2} \]

SVMs trainiert werden. Die Entscheidungsgrenzen ergeben sich aus der Kombination aller Klassifikatoren [web:38][web:60].
