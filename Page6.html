⚙️ DBSCAN: Hyperparameter-Suche

Bei DBSCAN sucht man typischerweise im zweidimensionalen Hyperparameterraum nach einer guten Kombination aus eps und min_samples.

Wichtig ist dabei, dass es keine universell richtige Einstellung gibt. Stattdessen hängt die Wahl von der Datenstruktur, dem Distanzmaß und der gewünschten Clusterauflösung ab.

from sklearn.cluster import DBSCAN

best_score = -1
best_params = None

for eps in [0.05, 0.10, 0.15, 0.20, 0.25]:
    for min_samples in [3, 5, 8, 10]:
        dbscan = DBSCAN(eps=eps, min_samples=min_samples)
        labels = dbscan.fit_predict(X)
        # hier z. B. Silhouette Score oder manuelle Bewertung verwenden
        if len(set(labels)) > 1:
            pass

Die DBSCAN-Klasse in sklearn besitzt bewusst keine .predict()-Methode, damit die Zuordnung neuer Datenpunkte flexibel gelöst werden kann.

Neue Punkte zu DBSCAN-Clustern zuordnen

Für neue Datenpunkte kann ein k-Nearest-Neighbors-Klassifikator auf den Core-Punkten des DBSCAN-Modells trainiert werden.

from sklearn.neighbors import KNeighborsClassifier

knn = KNeighborsClassifier(n_neighbors=50)
knn.fit(dbscan.components_, dbscan.labels_[dbscan.core_sample_indices_])

new_point_1 = [[-0.5, 0.5]]
new_point_2 = [[2, -0.5]]
print(knn.predict(new_point_1))
print(knn.predict(new_point_2))

Vergleich verschiedener Clustering-Verfahren

Im Vergleich zu k-means ist DBSCAN robuster gegenüber Ausreißern und erkennt nicht-konvexe Formen. Für realistische Daten ist daher häufig ein empirischer Vergleich sinnvoll.

Wenn die Struktur stark nichtlinear ist oder Rauschen eine wichtige Rolle spielt, ist DBSCAN oft die bessere Wahl als k-means.
